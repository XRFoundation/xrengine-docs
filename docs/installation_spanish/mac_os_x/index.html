Instalación en Mac OS X

  1. Ir a la raíz y ejecutar 
instalación de npm 
npm ejecute dev-docker 
npm ejecutar dev-reinit

O si está en un Mac basado en M1

(Recomendado) 1) Duplicar la aplicación de terminal y configurarla para que se ejecute en Rosetta 2) Ejecute lo anterior en Rosetta Terminal

(No recomendado) 

instalación de hilo

Esto se debe a que en los chips de Apple el nodo-darwin a veces no se instala correctamente y mediante el uso de hilo se arregla el problema.

  1. Tener docker iniciado en segundo plano y luego en el tipo de terminal 
npm ejecutar dev

Esto abrirá los scripts mariaDB y SQL en el docker e iniciará los servidores

  1. Para asegurarse de que su entorno está configurado y funcionando correctamente, vaya a https://localhost: 3000/location/default y debería poder caminar alrededor de una escena 3D vacía. 
Nota: Asegúrese de que está en Nodo > = 16 y tiene docker en ejecución.

Resolución de problemas

  • Si encuentra problemas en su terminal que dice que el acceso-denegado para el usuario server@localhost entonces puede utilizar este mandato 
brew services stop mysql
  • Si encuentra problema en su terminal que dice Se ha producido un error inesperado: " paquete de espacio de trabajo esperado mientras que el uso de hilado entonces usted puede utilizar este comando en su terminal 
Conjunto de políticas de hilado-versión 1.18.0

Como hilo > 1.18 a veces no funciona correctamente con lerna.
