Instalación

La puesta en marcha requiere solo unos pocos pasos, pero esto puede ser complicado, dependiendo de su plataforma y entorno actual. Por favor, siga las instrucciones para su entorno.

Lista de comprobación previa a la instalación

  • Clonar el repositorio
  • Instalar Node.js v16 o posterior
  • Instalar Python > =3,6 + PIP , C + +, y otras herramientas de construcción. Consulte la publicación Instrucciones de instalación de Mediasoup para obtener todos los detalles.
  • Instalar Docker (opcional pero muy recomendable)
  • (Opcional) Si NO está utilizando docker, instale MariaDB y Redis.

Ahora debería estar listo para seguir el Inicio rápido instrucciones.

Clonar el repositorio

Mucho ha cambiado durante el desarrollo, y nuestro monorapo ha conseguido bastante grande. Para evitar la clonación de todo, utilice este mandato: 

git clone https://github.com/XRFoundation/XREngine --depth 1

Asegúrese de que está en el nodo 16 o superior

Usted must tiene el nodo 16 o superior instalado.

Un gestor de versiones puede ser útil para esto:

  • Sólo nodeJS: NVM
  • Políglota: ASDF

Antes de ejecutar el motor, compruebe nodo -- versión Si utiliza una versión de nodo inferior a 16, actualice o nada funcionará. Usted sabrá que está teniendo problemas si intenta instalar en la raíz y está recibiendo errores de dependencia.

Docker es tu amigo

Usted no necesita usar Docker , pero hará que tu vida sea mucho más fácil. Si no desea utilizar Docker, deberá configurar mariadb y redis en su máquina. Puede encontrar credenciales en xrenina/scripts/docker-compose.yml

Inicio rápido

Si tienes suerte, esto sólo funcionará. Sin embargo, es posible que encuentre algunos problemas. Asegúrese de que ejecuta el nodo 16 y compruebe las dependencias. 

Path/to/xrenina 
cp .env.local.default .env.local 
instalación de npm 
npm ejecute dev-docker 
npm ejecutar dev-reinit 
npm ejecutar dev

Setup Elastic Search & grafana

La búsqueda elástica y la Grafana se lanzarán automáticamente con npm ejecutar dev .

Elasticsearch & Grafana se ejecutará en el puerto localhost 9200 y 5601, respectivamente.

Esto configurará y ejecutará automáticamente los contenedores de docker de Redis/MariaDB y las instancias de client/servidor/servidor de XRengine.

En un navegador, vaya a https://127.0.0.1:3000/location/default

El proceso de inicialización de la base de datos crea una ubicación vacía de prueba denominada 'test'. Se puede navegar a por ir a ' https://127.0.0.1:3000/location/default https://127.0.0.1:3000/location/default "

A partir de esta escritura, el cert proporcionado en el paquete xengine para uso local no está debidamente firmado. Los navegadores arrojarán advertencias acerca de ir a páginas inseguras. Usted debe ser capaz de decirle al navegador que lo ignore, por lo general haciendo clic en algún tipo de botón de 'opciones avanzadas' o enlace y luego algo a lo largo de las líneas de 'ir allí de todos modos'.

Chrome a veces no muestra una opción clickable en la advertencia. Si es así, sólo escriba badidea o estosinseguras cuando en esa página. No se introduce eso en la barra de direcciones o en un cuadro de texto, Chrome solo escucha pasivamente esos comandos.

Administración del sistema y del usuario

Puede administrar muchas características desde el panel de administración en https://localhost: 3000/admin

Cómo hacer que un usuario sea un administrador:

Crear un usuario en https://localhost: 3000/

Para localizar su ID de usuario: en la consola de herramientas de Chrome Dev, escriba copy (IDusuario) . Esto copiará su ID de usuario (como se muestra en la captura de pantalla adjunta). Pegue y ejecute el siguiente mandato en un shell ' nix (por ejemplo, Bash, ZSH):

npm ejecute make-user-admin -- -- id = {COPIED_USER_ID}

Ejemplo: npm ejecute make-user-admin -- -- id=c06b0210-453e-11ec-afc3-c57a57eeb1ac
