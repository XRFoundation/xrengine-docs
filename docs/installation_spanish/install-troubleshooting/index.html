Errores de certificado no válidos en el entorno local

A partir de esta escritura, el cert proporcionado en el paquete xengine para uso local no está debidamente firmado. Los navegadores arrojarán advertencias acerca de ir a páginas inseguras. Usted debe ser capaz de decirle al navegador que lo ignore, por lo general haciendo clic en algún tipo de botón de 'opciones avanzadas' o enlace y luego algo a lo largo de las líneas de 'ir allí de todos modos'.

Chrome a veces no muestra una opción clickable en la advertencia. Si es así, sólo escriba badidea o estosinseguras cuando en esa página. No se introduce eso en la barra de direcciones o en un cuadro de texto, Chrome solo escucha pasivamente esos comandos.

Permitir conexión de dirección de gameserver mediante la instalación de la Autoridad certificadora local

Para obtener instrucciones más detalladas, compruebe: https://github.com/FiloSottile/mkcert

Versión corta (común para el proceso de desarrollo en Ubuntu):

  1. sudo apt install libnss3-tools
  2. Instalar mkcert (Si no tienes brew, revisa la página: https://brew.sh/ )
  3. mkcert -- install
  4. navegar a ./certs carpeta
  5. mkcert -key-file key.pem -cert-archivo cert.pem localhost 127.0.0.1 :: 1

Permitir conexión http-server de archivo local con certificado no válido

Abra las herramientas de desarrollador en el navegador pulsando Ctrl + Mayús + i al mismo tiempo. Vaya a la pestaña 'Consola' y mire el historial del mensaje. Si hay errores rojos que dicen algo como GET https://127.0.0.1:3030/socket.io/? EIO=3&transport=polling&t=NXlZLTa net: :ERR_CERT_AUTHORITY_INVÁLIDO , a continuación, pulse con el botón derecho del ratón en dicho URL y, a continuación, seleccione 'Abrir en nueva pestaña' y acepte el certificado no válido.

Permitir conexión de dirección gameserver con certificado no válido

La funcionalidad de gameserver se aloja en una dirección distinta de 127.0.0.1 en el entorno local. La aceptación de un certificado no válido para 127.0.0.1 no se aplicará a esta dirección. Abra la consola dev para Chrome/Firefox pulsando Ctrl + Mayús + i de forma simultánea, e ir a las pestañas Consola o Red.

Si ve errores al no poder conectarse a algo como https://192.168.0.81:3031/socket.io/?location=<foobar> , pulse con el botón derecho del ratón en ese URL y ábralo en una nueva pestaña. Debe volver a obtener una página de aviso sobre un certificado no válido y de nuevo debe permitirlo.

AccessDenied connecting to mariadb

Asegúrese de que no tiene otra instancia de mariadb ejecutándose en el puerto 3306 

lsof -i: 3306

En Linux, también puede comprobar si los procesos se están ejecutando en el puerto 3306 con sudo netstat -utlp | grep 3306 La última columna debe ser como <ID>/<algo Puede eliminar cualquier proceso en ejecución con sudo kill <ID>

Error: escucha EADDRINUSE :::3030

comprobar qué proceso está utilizando el puerto 3030 y matar 

Nodo kill -9

O 

lsof -i: 3030 
kill -3 <proccessIDfromPreviousCommand>

'Error CORS' en terminal

Intente acceder a la página utilizando https://localhost: 3000 en lugar de https://127.0.0.1:3000

Pantalla en blanco predeterminada

Intente escribir "estosinseguras" o "iknowwhatiamdoing" luego volver a cargar

¿Error de carga de gameserver o recurso?

Abra la consola dev, pulse el enlace GET en la nueva pestaña y acepte el certificado escribiendo thissafe " o "iknowwhatiamdoing" a continuación, recargar página original

¿Cuelga en la página principal?

Intente escribir lo siguiente en el terminal, en el directorio /packages/server 

npm ejecute dev-reinit-db

Para iniciar sesión como admin

En la herramienta de desarrollo de Chrome userId Esto mostrará el ID de usuario. Copie este ID de usuario como serie, ejecúelo como siguiente mandato en shell: 

npm ejecute make-user-admin -- -- id = {COPIED_USER_ID}

Ejemplo 

npm ejecute make-user-admin -- -- id=c06b0210-453e-21ws-afc3-c97a57eeb1ac

Para instalar un nuevo paquete para los componentes de reacción del editor en monorepo

Tipo en terminal 

 npm i <packagename> -w @xrengine/editor

Para volver a llenar la base

Matar primero el servidor y, a continuación, escribir en terminal 

cd packages/server & & npm ejecute dev-reinit-db

DB no sembrando rutas (por ejemplo, Error: No hay ningún proyecto instalado-póngase en contacto con el administrador del sitio)

Probar 

npm ejecutar dev-reinit

o 

Contenedor de docker stop xengine_db 
docker contenedor rm xengine_db 
Poda de contenedor de docker 
npm ejecute dev-docker 
npm ejecutar dev-reinit

'TypeError: No se puede leer la propiedad' position 'de undefined' al acceder a /location/home

A partir de esta escritura, hay un error con la ubicación de prueba preclasificada por defecto. Vaya a /editor/projects y abra el proyecto 'Test'. Guarde el proyecto y el error debe desaparecer.

¿Problemas extraños con su base de datos?

Probar 

npm ejecutar dev-reinit

o si en ventanas 

npm ejecutar dev-reinit-windows
