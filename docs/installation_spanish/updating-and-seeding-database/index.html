Actualización de modelos de base

Las herramientas para actualizar automáticamente la base de datos en función de los cambios en los modelos se incluyen en sequelize.ts . La mayor parte se controla estableciendo la variable de entorno PREPARE_DATABASE=true .

Si se establece, la configuración de la base de datos iterará a través de los campos de cada modelo y tratará de coincidir cada uno con una columna. Si no puede encontrar una columna existente, entonces se producirá una de las dos cosas:

  • Si el modelo tiene un valor oldColumna establecido en la definición de campo/foreignKey, y esa columna antigua existe, la columna antigua se redenominará al nombre actual del campo de modelo

El siguiente es un ejemplo de user.model.ts donde el campo inviteCode se redenominará a = código , y la asociación que fue llamada channelInstanceId se redenomina a instanceChannelId . Tenga en cuenta el oldColumna en cada uno de ellos.

  • Si el modelo no tiene oldColumna , o lo hace y la columna antigua no existe, entonces añadirá una nueva columna

Si la columna existe, el modelo actual para esa columna se aplica a la columna

Siembra

La siembra ocurre si FORCE_DB_REFRESH=true o PREPARE_DATABASE=true . Cada plantilla de semilla se repite y la base de datos se comprueba para ver si existe ese valor de plantilla. Las plantillas de semilla con ID se comprueban a través de una .get (<id>) , las plantillas de semillas sin un ID se comprueban a través de un .find ({consulta: <template> }) (Las tablas de configuración sólo comprueban si hay una fila presente, y asume que su presencia es indicativa de ser sembrada ya). Si no se encuentra una plantilla de semilla, se inserta en la base de datos.
