running-on-static-IP

Siga estos pasos para ejecutar el motor en una IP estática en lugar de localhost. En la mayoría de los casos, debe ser capaz de simplemente acceder al motor utilizando la IP pública asignada a su dispositivo, pero si se ejecuta en cualquier problema o si está ejecutando la pila en WSL2, puede consultar las siguientes direcciones.

Ejecución en IP estática bajo WSL2

  1. Sustituya todos los valores de localhost por la IP estática en la que desea ejecutar la pila en el archivo .env.local.
  2. Abra un terminal de powershell como administrador. Y ejecute el script wsl2-port-forwarding.ps1 presente en el directorio /scripts. Nota: Asegúrese de que todos los puertos necesarios están presentes en la matriz de puertos del script wsl2-port-forwarding.ps1.
  3. Y ahora sólo ejecutar el motor como lo haría normalmente y todo debe ser accesible sobre la IP estática.
  4. Si obtiene algún error relacionado con localhost: 8642, asegúrese de que ninguno de los activos de la escena se haya guardado en la vía de acceso de host local. Si a continuación, se sustituye localhost por la IP estática en la vía de acceso del activo correspondiente.
