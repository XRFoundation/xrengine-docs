Configuración avanzada

Si desea configurar las instancias de docker de XREngine, el cliente, el servidor y/o el servidor de juegos manualmente, siga estas instrucciones. La configuración avanzada es recomendable para todos los usuarios, con el fin de entender más sobre todo lo que está pasando.

1. Instale sus dependencias 

Path/to/xrenina 
instalación de npm 
npm ejecute dev-docker 
npm ejecutar dev-reinit

No debe tener que utilizar sudo en ningún caso.

¿Error con mediasoup? https://mediasoup.org/documentation/v3/mediasoup/installation/

2. Asegúrese de tener una base de datos mysql instalada y en ejecución -- nuestra recomendación es Mariadb.

Hemos proporcionado un contenedor de docker para una configuración fácil: 

cd scripts & & sudo bash start-db.sh

Esto crea un contenedor Docker de mariadb llamado xengine_db. Debe tener instalado docker en la máquina para que funcione este script. Si no tiene Docker instalado y no desea instalarlo, tendrá que crear manualmente un servidor MariaDB.

El nombre de usuario predeterminado es 'server', la contraseña predeterminada es 'password', el nombre de base de datos predeterminado es 'xengine', el nombre de host predeterminado es '127.0.0.1' y el puerto predeterminado es 3306 .

¿Ver errores al conectarse a la base de datos local? Intente apagar el cortafuegos local.

3. Abra una nueva pestaña e inicie los Agones sidecar en modo local 

scripts de CD 
sudo bash start-agones.sh

También puede ir al proveedor/agones/ y ejecutar

./sdk-server.linux.amd64 -- local

Si está utilizando una máquina Windows, ejecute

sdk-server.windows.amd64.exe -- local

y para mac, ejecutar

./sdk-server.darwin.amd64 -- local

4. Inicie el servidor en modalidad de semilla de base de datos

Es necesario que varias tablas de la base de datos se presiembren con valores predeterminados. Ejecutar npm ejecutar dev-reinit o si en ventanas npm ejecutar dev-reinit-windows Después de varios segundos, no debería haber más registro. Algunas de las líneas finales deberían leer así: 

Servidor preparado 
Ejecutando (valor predeterminado): SET FOREIGN KEY_CHECKED = 1 
Servidor EXIT

En este punto, la base de datos ha sido sembrada.

5. Configuración del servidor de archivos local

Si el archivo .env.local que tiene tiene la línea STORAGE_PROVIDER=LOCAL el editor de escenas guardará los componentes, modelos, escenas, etc. localmente (en lugar de almacenarlos en S3). Tendrá que iniciar un servidor local para servir estos archivos y asegurarse de que .env.local tiene la línea LOCAL_STORAGE_PROVIDER="LOCALHOST: 8642 " . En una nueva pestaña, vaya a Paquetes/servidor y ejecutar npm ejecutar servicio-archivos locales . Esto se iniciará http-server para servir archivos de paquetes/server/upload localhost: 8642 . Es posible que tenga que aceptar el certificado autofirmado no válido para él en el navegador; consulte 'Allow local file http-server connection with inválido certificate' below.

6. Abra dos/tres pestañas separadas e inicie el servidor de API, gameserver y cliente

En /packages/server, ejecute npm ejecutar dev que lanzará el servidor de la api, el servidor de juegos y el servidor de archivos. Si no está utilizando gameservers, puede ejecutar en su lugar npm ejecute dev-api-server en el servidor de api. En la pestaña final, vaya a /packages/client y ejecute npm ejecutar dev . Si usted está en las ventanas que necesita utilizar npm ejecute dev-windows en lugar de npm ejecutar dev .

7. En un navegador, vaya a https://127.0.0.1:3000/location/default

El proceso de inicialización de la base de datos crea una ubicación vacía predeterminada denominada 'default'. Se puede navegar a por ir a ' https://127.0.0.1:3000/location/default https://127.0.0.1:3000/location/default " . En cuanto a esta escritura, el cert proporcionado en el paquete XREngine para uso local no está debidamente firmado. Puede crear certificados firmados y sustituir los predeterminados, pero la mayoría de los desarrolladores sólo ignoran los avisos. Los navegadores arrojarán advertencias acerca de ir a páginas inseguras. Usted debe ser capaz de decirle al navegador que lo ignore, por lo general haciendo clic en algún tipo de botón de 'opciones avanzadas' o enlace y luego algo a lo largo de las líneas de 'ir allí de todos modos'.
