Instrucciones de Docker antiguas

Puede acelerar localmente el uso de docker, si no tiene el nodo instalado o simplemente desea probar lo último.

Obtener dirección IP local

Utilice una herramienta como ifconfig para obtener su dirección IP local.

Iniciar bases de datos locales 

cd  scripts 
docker-componer

Cuando se detiene el registro, indica que las bases de datos se han creado y se están ejecutando.

Ctrl + c de eso, a continuación, de los scripts ejecutados ./start-all-docker.sh (Esto debe ejecutarse cada vez que inicie la máquina de nuevo)

Crear la imagen

Crear una carpeta vacía en la raíz llamada Proyecto-paquete-jsons y ejecute el mandato siguiente para crear: 

DOCKER_BUILDKIT = 1   docker  build -t xrenine -- build-arg   MYSQL_USER = servidor   \ 
 -- arg build-arg   MYSQL_PASSWORD = contraseña -- build-arg   MYSQL_HOST = 127.0 .0.1   \ 
 -- arg build-arg   MYSQL_DATABASE = xrenina -- arg   MYSQL_PORT = 3304   \ 
 -- arg build-arg   VICE_SERVER_HOST = localhost -- build-arg   VITE_SERVER_PORT = 3030   \ 
 -- arg build-arg   VITE_GAMESERVER_HOST = localhost -- build-arg   VITE_GAMESERVER_PORT = 3031   \ 
 -- arg build-arg   VITE_LOCAL_BUILD = true -- build-arg   CACHE_DATE = " $( fecha ) "  -- red = "host"   .

Ejecute el servidor para inicializar la base de datos, espere un par de minutos y, a continuación, suprímalo 

docker  run -d -- name server -- env-file .env.local.default -e   "SERVER_MODE = api"  -e   "FORCE_DB_REFRESH = true"  -- red   host  xrenina 
docker  servidor de registros -f 
-Espera   para  la línea   "Servidor listo" ,   entonces  Ctrl + c fuera de los registros- 
docker  Servidor de detención 
docker  Contenedor   rm  servidor

Ejecutar las imágenes 

docker  run -d -- name serve-local -- env-file .env.local.default -e   "SERVER_MODE = serve-local"  -- red   host  xrenina 
docker  run -d -- name server -- env-file .env.local.default -e   "SERVER_MODE = api"  -e   "GAMESERVER_HOST = <dirección IP local"  -- red   host  xrenina 
docker  run -d -- name client -- env-file .env.local.default -e   "SERVER_MODE = cliente"  -- red   host  xrenina 
docker  run -d -- name world -- env-file .env.local.default -e   "SERVER_MODE = tiempo real"  -e   "GAMESERVER_HOST=<local IP address>"  -- red   host  xrenina 
docker  run -d -- name canal -- env-file .env.local.default -e   "SERVER_MODE = tiempo real"  -e   "GAMESERVER_HOST=<local IP address>"  -e   "GAMESERVER_PORT=3032"  -- red   host  xrenina

Suprimir contenedores, si desea ejecutar una nueva compilación, o simplemente deshacerse de ellos 

docker  servicio de parada de contenedor-local 
docker  Contenedor   rm  servicio-local 
docker  Servidor de detención 
docker  Contenedor   rm  servidor 
docker  Cliente de detención 
docker  Contenedor   rm  cliente 
docker  parada del contenedor 
docker  Contenedor   rm  mundo 
docker  Canal de detención 
docker  Contenedor   rm  canal
